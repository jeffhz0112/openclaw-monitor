## 🚀 快速部署到 Unraid

### 方法一：使用快速部署脚本（推荐）

在 Mac mini 上运行：

```bash
cd /path/to/openclaw-monitor
bash quick-deploy.sh
```

脚本会自动：
1. 检查 SSH 连接
2. 上传文件到 NAS
3. 构建 Docker 镜像
4. 启动监控容器

### 方法二：手动部署

1. **上传文件到 NAS**
   ```bash
   scp -r openclaw-monitor root@192.168.31.161:/mnt/user/appdata/
   ```

2. **SSH 登录到 NAS**
   ```bash
   ssh root@192.168.31.161
   ```

3. **进入项目目录**
   ```bash
   cd /mnt/user/appdata/openclaw-monitor
   ```

4. **构建并启动容器**
   ```bash
   docker build -t openclaw-monitor .
   docker run -d \
     --name openclaw-monitor \
     --restart unless-stopped \
     -p 8080:8080 \
     --env-file .env \
     -v $(pwd)/data:/app/data \
     openclaw-monitor
   ```

## 🌐 访问监控界面

部署成功后，浏览器访问：
```
http://192.168.31.161:8080
```

## ⚙️ 配置 Telegram 通知（可选）

如果需要状态变化时收到 Telegram 通知：

1. 参考 `TELEGRAM_SETUP.md` 创建 Bot 并获取 Token 和 Chat ID
2. 编辑 NAS 上的 `.env` 文件：
   ```bash
   ssh root@192.168.31.161
   cd /mnt/user/appdata/openclaw-monitor
   nano .env
   ```
3. 填入以下信息：
   ```
   TELEGRAM_BOT_TOKEN=你的_Bot_Token
   TELEGRAM_CHAT_ID=你的_Chat_ID
   ```
4. 重启容器：
   ```bash
   docker restart openclaw-monitor
   ```

## ❓ 问题排查

### 无法访问监控界面

1. 检查容器是否运行：
   ```bash
   docker ps | grep openclaw-monitor
   ```

2. 查看容器日志：
   ```bash
   docker logs openclaw-monitor
   ```

3. 检查端口占用：
   ```bash
   netstat -tulpn | grep 8080
   ```

### 无法连接到 Mac mini

1. 确认 Mac mini 和 NAS 在同一网络
2. 确认 OpenClaw 正在运行
3. 检查 Mac mini 防火墙设置
4. 从 NAS ping Mac mini：
   ```bash
   ping 192.168.31.69
   ```

### Telegram 通知不工作

1. 检查 `.env` 文件中的 Token 和 Chat ID
2. 确认 Bot 有权限发送消息
3. 查看日志中的错误信息
